# Changelog

Clipton の `0.1.9` 以降の主な変更点をまとめます。

このファイルは今後のリリースごとにメンテする前提です。新しい変更はまず `Unreleased` に追記し、リリース時に該当バージョンへ移してください。

> Note: このリポジトリには現時点で Git tag / GitHub Releases がないため、バージョン境界は `Clipton.WinUI.csproj` と `Package.appxmanifest` の version bump コミットから推定しています。

## Unreleased

なし。

## 0.1.19 - 2026-07-10

### Changed

- アプリと Store パッケージのバージョンを `0.1.19` / `0.1.19.0` に更新しました。
- 履歴エクスポートがメモリ上に読み込まれていない暗号化履歴も含めるようになり、保存データを完全に読めない場合は不完全なバックアップを作らずエラーにするようにしました。
- Quick menu と画像プレビューのウィンドウサイズ・配置をモニターごとの DPI に合わせるようにしました。
- Quick menu と設定画面で Windows のハイコントラスト配色を使い、主要な操作項目に UI Automation の名前と識別子を追加しました。
- Store パッケージに 7 言語の UI リソースをすべて含め、CI でも各言語の宣言を検査するようにしました。

### Fixed

- 履歴インポートの外部 ID を新しい GUID に置き換え、重複 ID や不正なファイルパスが履歴保存へ持ち込まれないようにしました。
- 部分読み込み中に削除した履歴が次回保存で復活する問題と、保存失敗時に以前の manifest が参照する payload を先に削除する問題を修正しました。
- クリップボード書き込みに失敗した場合や、元の貼り付け先ウィンドウを確認できない場合に、古いクリップボード内容へ `Ctrl+V` を送らないようにしました。
- 暗号化エクスポートの PBKDF2 反復回数に許容範囲を設け、細工されたファイルによる過大な鍵導出処理を防ぎました。
- ユーザー定義のマスク正規表現がタイムアウトした場合に、内容を表示せず保護対象として扱うようにしました。
- Rich quick menu のテキストフィルターが実際の種別ラベルと一致せず、項目を正しく絞り込めない問題を修正しました。
- ファイル履歴の参照先が削除済みの場合、クリップボード更新を成功扱いにしないようにしました。
- 終了処理を複数回呼んでも安全にし、終了時の最終履歴保存とウィンドウ破棄を確実に行うようにしました。
- 設定ファイルの読み取り失敗、null コレクション、保存失敗時の一時ファイル残留を処理するようにしました。

## 0.1.18 - 2026-07-10

### Changed

- アプリと Store パッケージのバージョンを `0.1.18` / `0.1.18.0` に更新しました。
- Store/スタートメニュー向けのパッケージアイコンの余白を調整し、表示サイズを大きくしました。
- 履歴検索に 150ms のデバウンスを追加し、検索判定中に全候補の表示用 Preview を構築しないようにしました。
- `settings.json` の保存を一時ファイル経由のアトミック書き込みに変更しました。

### Fixed

- itemized 履歴の manifest が存在しない `.dat` を参照している場合に、診断ログを残して manifest を補正するようにしました。
- ファイル貼り付け対象が削除済みだった場合に、無反応にならないよう診断ログへ記録するようにしました。
- 履歴検索の未知 `key:value` トークンがコロン込み完全一致にならず、通常語検索へフォールバックするようにしました。
- WPF 版でグローバルホットキー登録に失敗した場合、トレイ通知でユーザーへ知らせるようにしました。
- 暗号化エクスポートの復号時に、ファイル内の PBKDF2 反復回数を使うようにし、将来の反復回数変更後も旧エクスポートを読めるようにしました。

## 0.1.17 - 2026-07-06

### Added

- About タブで Microsoft Store の更新有無を確認し、更新がある場合は About タブにバッジ表示してユーザー操作で更新を開始できるようにしました。
- About タブの Store 更新カードから CHANGELOG を開けるようにしました。
- Store 更新確認中のボタン表示と、確認後の最新/更新あり結果表示を分かりやすくしました。
- 英語、日本語に加えて、ドイツ語、スペイン語、フランス語、韓国語、中国語（簡体字）の UI リソースを追加しました。
- Store パッケージのアプリ名/説明を `ms-resource` 化し、各 UI 言語の `Resources.resw` を追加しました。
- Quick menu のライト/ダークテーマ用パレットを追加し、通常メニュー、リッチメニュー、画像プレビューの配色をテーマに追従させました。
- Quick menu のテーマパレット回帰テストと、多言語 UI 文字列の回帰テストを追加しました。
- Quick menu とトレイ右クリックメニューから単語登録を直接開ける導線を追加しました。

### Changed

- README、Store 掲載ドラフト、Store アセット計画、Store 準備チェックリスト、永続化パフォーマンス文書を最新の WinUI アプリ仕様に合わせて更新しました。
- README のローカルデータ説明を、現在の `snippets.dat` と v5 itemized history 形式に合わせて更新しました。
- アプリと Store パッケージのバージョンを `0.1.17` / `0.1.17.0` に更新しました。

### Fixed

- Quick menu の画像プレビュー、フォルダ読み込み表示、リッチメニューのフィルタ/戻る/閉じる/相対時刻などに残っていた固定英語表記をローカライズしました。
- ライトテーマ指定時に WinUI の通常/リッチ Quick menu が暗色固定で表示される問題を修正しました。
- 不正なスニペット日付書式やファイルパス変数で貼り付け処理が例外終了しないようにしました。
- 履歴保存・読込、manifest 暗号化、画像一時ファイル、クリップボード画像変換、PIN ロック試行回数、フック解除などの堅牢化を行いました。
- 機密情報マスク処理の順序依存、電話番号の過剰検出、サロゲートペア切断、Quick menu ショートカット判定などを修正しました。

## 0.1.15 - 2026-06-22

### Added

- Quick menu、履歴ページ、登録単語ページで履歴や登録単語を表示する前に PIN を要求できるロックを追加しました。
- PIN を忘れた場合に履歴と登録単語の削除確認つきで PIN ロックを初期化できる操作を追加しました。
- 履歴や登録単語のエクスポート直前に、暗号化されない JSON ファイルとして保存されることを確認し、PIN ロック有効時は PIN 再入力を求めるダイアログを追加しました。
- Quick menu の詳細操作をキーボードで選択して Enter する E2E 検証を追加しました。
- 詳細設定から Quick menu の詳細メニュー表示項目を選べるようにしました。
- 設定画面に詳細設定タブを追加しました。

### Changed

- デフォルトのグローバルホットキーを `Shift+Alt+V` に変更しました。
- Quick menu のショートカットヒントをデフォルト非表示にしました。
- サイドバー折りたたみ時はグローバルホットキー表示を隠すようにしました。
- テストプロジェクトのテスト基盤依存を更新し、既知の脆弱な推移的依存を解消しました。
- アプリと Store パッケージのバージョンを `0.1.15` / `0.1.15.0` に更新しました。

### Fixed

- Quick menu のショートカットヒントと取得時刻の表示設定が、既存のメニューウィンドウ再利用時にも即時反映されるようにしました。
- Rich quick menu でマウスホバーによる選択変更だけでは自動スクロールしないようにしました。
- Rich quick menu の詳細メニューで左キーを押したとき、元の項目へ戻れるようにしました。

## 0.1.12 - 2026-06-17

### Added

- README に Clipton の概要説明画像を追加しました。
- 履歴の由来メタ情報を保持し、quick menu と履歴画面で参照できるようにしました。
- Quick menu から履歴/登録語をクイック編集して、そのまま貼り付けできるようにしました。
- Quick menu のトップレベル項目を数字キーで即貼り付け、Space でプレビュー、E で編集できるショートカットを追加しました。
- 画像プレビューをキーボード操作でも閉じやすいように、閉じるボタンを追加しました。

### Changed

- デフォルトのグローバルホットキーを `Ctrl+Alt+V` に変更しました。
- 起動時にホットキーが競合した場合、`Ctrl+Alt+Space`、`Ctrl+Shift+V` の順にプリセットへフォールバックするようにしました。
- Quick menu の default / rich ウィンドウを表示ごとに破棄せず、キャッシュして再利用するようにしました。
- Quick menu を開く直前に clipboard を短時間だけ更新し、直前にコピーした内容が履歴に反映されやすくしました。

### Fixed

- 永続履歴の遅延ロード時に、`History` の変更と列挙を UI thread に寄せ、background load と UI 更新の競合を避けるようにしました。
- Quick menu を再オープンした直後に古い dismiss cleanup が走って、新しい menu まで閉じてしまうケースを防ぎました。
- Default quick menu の初回表示位置を安定化し、初回起動時に上へ戻される挙動を抑えました。
- Quick menu 起動直後に下キーを押した場合、遅延フォーカス復元が選択位置を先頭へ戻さないようにしました。

## 0.1.11 - 2026-06-12

### Added

- Quick menu に launcher-style のインクリメンタル検索を追加しました。
- Quick menu の keyboard navigation を検証する E2E スクリプトを追加しました。
- 既に起動中の Clipton がある場合に既存インスタンスを利用する単一インスタンス制御を追加しました。

### Changed

- Quick menu の検索 UI を rich window に統合し、ライブフィルタリングできるようにしました。
- Quick menu の search toolbar button が検索中に active 表示になるようにしました。
- default quick menu で `>` chevron をフォルダ専用にし、paste options は `...` として区別しやすくしました。
- Quick menu フォルダ内で Right key から paste options を開けるようにしました。
- Clipboard capture と履歴永続化を UI thread から分離し、操作中の固まりを軽減しました。
- 履歴一覧の行を再利用し、更新時に毎回全体を作り直さないようにしました。
- Clipboard text extraction を `Clipton.Core` 側に共通化し、テストを追加しました。

### Fixed

- Quick menu の keyboard focus が不安定になる問題を修正しました。
- Rich quick menu の Left / Right folder navigation を修正しました。
- scroll による hover が rich menu の keyboard selection を奪う問題を修正しました。
- Window size と chrome の設定順を調整し、quick menu 表示時の flash を避けるようにしました。
- UI thread に `DispatcherQueueSynchronizationContext` を設定し、WinUI 周りの async 継続を安定化しました。
- Quick menu の async path と clipboard retry exhaustion 周りを堅牢化しました。
- `ClipboardBridge` tests から real clipboard 依存を取り除きました。
