Clipton Privacy Policy

Last updated: 2026-05-21

日本語

Clipton は、直近のクリップボード履歴を表示し、選択した項目を貼り付けるためにクリップボード内容を読み取ります。内容にはテキスト、リッチテキスト、HTML、画像、ファイルパスが含まれる場合があります。

暗号化履歴が有効な場合、履歴はユーザーのアプリデータフォルダーにローカル保存され、Windows のユーザースコープ DPAPI で保護されます。設定と登録メッセージもローカルに保存されます。

Clipton は、クリップボード内容、設定、登録メッセージ、利用状況データをサーバーへアップロードしません。現在のバージョンには、クラウド同期、テレメトリ、クラッシュレポート、広告 SDK、第三者へのデータ共有はありません。

ユーザーは、記録の一時停止、履歴項目の個別削除、履歴の全削除、暗号化ローカル履歴の無効化、Windows 起動時の開始を設定から制御できます。

English

Clipton reads clipboard content so it can show recent clipboard history and paste selected items. Clipboard content may include text, rich text, HTML, images, and file paths copied from other apps.

When encrypted history is enabled, clipboard history is stored locally in the user's application data folder and protected with Windows user-scoped DPAPI. Settings and snippets are also stored locally.

Clipton does not upload clipboard content, settings, snippets, or usage data to any server. The current version does not include cloud sync, telemetry, crash reporting, advertising SDKs, or third-party data sharing.

Users can pause capture, delete individual history items, clear all history, disable encrypted local history, and control startup behavior from settings.
